# Intranet (Local File Server)

Serve files from a local directory over HTTP. Perfect for viewing HTML outputs, documentation, or any web content generated by other skills.

## Default Root Directory

- `~/clawd/intranet/`

Override with:
- `--dir <path>` or env `INTRANET_DIR`

## Commands (via scripts/intranet.py)

- Start server:
  - `python3 skills/intranet/scripts/intranet.py start`
  - Optional: `--host <host>` (default: 0.0.0.0)
  - Optional: `--port <port>` (default: 8080)
  - Optional: `--dir <path>` (default: ~/clawd/intranet)

- Check status:
  - `python3 skills/intranet/scripts/intranet.py status`

- Stop server:
  - `python3 skills/intranet/scripts/intranet.py stop`

## Plugin Support

Other skills can add content to the intranet by creating symlinks in the root directory:

```bash
# Example: Add banker web interface
ln -s ~/banker ~/clawd/intranet/banker

# Example: Add custom documentation
ln -s ~/my-docs ~/clawd/intranet/docs
```

Any symlinked directory will be served at `http://localhost:8080/<name>/`

## Notes

- The server runs as a background daemon process
- Process ID is stored in `~/.intranet.pid`
- Server configuration is stored in `~/.intranet.conf`
- Access the server at `http://localhost:8080/` (or configured host/port)
